The Axiom · 안전한 적용

diff로 안전하게 개발자가 직접 확인·적용

AI 코드 도구의 가장 큰 불안은 "통째로 갈아엎어 망가뜨리는 것"입니다. Axiom은 파일을 다시 쓰지 않고 바뀐 부분만 고쳐서, 모든 변경을 사람이 검토하고 되돌릴 수 있는 diff로 제안합니다.

domains/member/MemberList.tsx multi-patch
11 const { data } = useApi('/members')
12- return <div>{data.map(r => r.name)}</div>
12+ return <Table rows={rows} cols={cols} />
⋮선택 영역 밖은 그대로 — 변경이 번지지 않음
27 const rows = $util.array.toRows(data)
28+ const cols = $util.table.cols(memberSchema)
✓ 2곳을 부분만 수정 · 나머지 라인은 손대지 않음
⊟
부분 수정 multi-patch

한 번의 응답으로 여러 곳을 부분만 정확히 고칩니다. 파일을 통째로 다시 쓰지 않습니다.

라인 단위 편집 line edit

바뀐 줄만 지정해 고쳐서, 출력도 적고 변경 범위도 작게 유지합니다.

⛉
안전장치 grounded retry · 범위 가드

위치를 못 찾으면 억지로 적용하지 않고 코드 재확인 → 사용자 선택으로. 선택 영역 밖 변경은 차단합니다.

⤓
오프라인 폴백 offline fallback

LLM 서버가 응답하지 않아도 scaffold 기반 스텁으로 작업이 끊기지 않습니다.

결과적으로 모든 변경이 검토 가능하고 되돌릴 수 있는 diff로 들어옵니다 — AI를 믿고 쓸 수 있는 이유입니다.
