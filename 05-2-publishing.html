디자인 시스템 · 협업 워크플로우

퍼블리셔 전용 작업 공간 src/publishing/

퍼블리셔가 디자인 토큰 + Tailwind로 컴포넌트를 작성하고 Storybook으로 팀에 공유합니다. 개발자와 퍼블리셔의 협업 경계를 깔끔하게 나눕니다.

퍼블리셔✎
src/publishing/{domain}/
  • 디자인 토큰 클래스로 작성
  • Tailwind 유틸 + cn()
  • 별도 CSS 파일 없음
className={cn('rounded-xl border', 'bg-brand-500 shadow-theme-sm')}
Storybook 공유
팀 프리뷰▦
npm run storybook
  • 컴포넌트 실시간 미리보기
  • Design Tokens 문서 동봉
  • 리뷰 · 피드백 공유
핸드오프
개발자⌗
src/domains/ · src/shared/
  • 라우터에 등록 · 연결
  • 비즈니스 로직 결합
  • API · 상태 연동
핸드오프 기준 — 작성한 위치에서 운영 위치로 이동
publishing/pages/→domains/[name]/pages/
publishing/components/→shared/ · domains/[name]/components/

경계가 분명 : 퍼블리셔는 토큰·마크업·스타일에 집중하고, 개발자는 라우팅·로직·연동을 담당합니다.
