## 프론트앤드 개발 생산성을 위한 Scaffole와 Axiom을 소개하겠습니다.
* 먼저 소개 영상을 보겠습니다.



## 그럼 이게 왜 필요한지 알아보겠습니다.

## SI 프로젝트를 투입하면 매번 반복되는 초기작업이 있습니다. 
* 프론트앤드를 기준으로 했을 때 환경설정 부터, 폴더구조, 공통모듈, UI 컴포넌트 작업  등 누군가는 항상 작업해야합니다.

## 화면 작업도 마찬가지 입니다.
* 화면에 필요한 비즈니스 로직 외, 부가적인 작업때문에 소스코드의 품질 저하는 물론 개발 시간도 부족할 수 있습니다.




## 그래서 필요한 건, 이미 다 갖춰진 Scaffold같은 기반 코드 입니다.

## Scaffold를 적용했을 때, 프론트앤드 초기 작업을 획기적으로 줄일 수 있을 것입니다.

## 화면작업을 할 때도 Scaffold를 도입하면, 개발자는 비즈니스 로직에 좀 더 집중하여 개발할 수 있을것입니다.




## 그래서 react-app-scaffold를 만들었습니다.

## react-app-scaffold는 
* React와 TypeScript, Vite 기반으로 만들어졌습니다.
* 그리고 도메인 기반 폴더 구조로 만들어져 있어서, 여려명의 개발자들이 한꺼번에 개발하더라도 최대한 충돌을 피할 수 있게 구성되어있습니다.
* 그리고 UI 컴포넌트, 각종 유틸리티, 인터렉티브 데모, 코드예제, 가이드 까지 모두 제공합니다.

## 개발 폴더 구조에 대해 설명드리겠습니다.
* 예를 들어 ... (A개발자는 Order업무, B개발자는 Payment업무...)
* 업무 폴더로 구분되어 있어서 충돌이 발생하지 않습니다.
* 새로운 업무가 추가되거나 삭제하기도 쉽습니다.
*  그럼에도 불구하고 부득이하게 다른 업무 영역의 모듈을 참조할 수 밖에 없는 경우가 생길 수도 있습니다.
* 이럴때는 shared 폴더를 통해서 공유 가능합니다.


## UI 컴포넌트...


## 개발자 편의를 제공하기위한 $UI 객체를 만들었습니다.
* 현재는 alert과 confirm만 있는데, 추후 dialog나 필요한 UI기능을 더 추가할 예정입니다.
* 예를 들어 기존에는 alert을 사용하기 위해서 .......
* 하지만 $ui객체를 사용하여 alert를 사용하면 이렇게 간단하게 사용할 수 있습니다. 이 부분은 데모 때 보여드리겠습니다.


## 유틸리티 사용법도 마찬가지 입니다.
* 이렇게 $util객체를 사용하면 간단하게 유틸리티 함수를 사용할 수 있습니다.
* 그리고 vscode에서는 자동완성을 제공해줘서 훨씬 더 사용하기 편리합니다.


## 화면작업할 때, 가장 많이 활용하는 것이 rest api 호출것과, 결과 데이트를 활용하는것일겁니다.
* Scaffold는 useApi 훅을 제공해서 데이터 처리에 필요한 여러가지 기능을 함께 제공합니다.
* 예를 들어서 로딩처리라던지, 에러처리, refetch 와같은 많은 기능을 제공합니다.
* 기존 방식으로 보면 이런 기능들은 일일이 다 작업 해줘야 하는 기능입니다.


## 개발자에게 최대한 편의를 제공하기 위하여 실제로 동작하는 인터렉티브 데모를 제공합니다.
* 이렇게 실제로 동작하는 기능을 확인하고, 예제 코드를 바로 카피에서 사용할 수 있습니다.


## 그리고 좀 더 자세한 가이드는 이렇게 사이트 형태로 개발 가이드를 따로 제공합니다.


## 디자인 토큰 시스템도 내장되어 있습니다.
* 퍼블리셔는 처음에 json으로 토큰을 작성하고, 이것을 빌드한 후에, css파일로 생성되면, 개발자는 그것을 화면에서 활용할 수 있습니다.

## 실제로 작업하는 순서를 보여드리면 ... 순서진행...

## 퍼블리싱된 파일은 publishing 폴더에 모두 작업하게 됩니다.
* 이렇게 작업된 파일은 Storybook으로도 작성될 수 있게 구성되어있습니다.
* 개발자는 storybook을 활용할 수도 있고, 실제 작업된 퍼블 파일도 사용하게됩니다.

## 이렇게 react-app-scaffold는 UI컴포넌트 부터, 폴더구조까지 여러가지 기능이 포함된 하나의 React 패키지입니다.
* Scaffold를 도입하면, 개발 초기에 빠르게 시작할 수 있고, 효율적인 개발과 품질을 보장할 수 있습니다.

## React외에도 Vue.js, Next.js, 그리고 마이크로 프론트앤드 Scaffold까지 계속해서 개발 할 예정입니다.
* 사실 이렇게 Scaffold만 하더라도 프론트앤드 개발 생산성에, 많은 도움이 될거라고 생각합니다.

## 최근에는 AI가 많이 활용되고 있습니다.
* 그래서 혹시 프론트앤드 개발에도 AI를 활용할 수 없을까 고민해봤습니다.

## Scaffold와 AI를 함께 활용하면 프론트앤드 개발 생성성에 많은 도움이 될거라고 생각됩니다.
