## 3. SI 프로젝트 투입 시 매번 똑같은 작업 반복

우리가 SI 프로젝트를 처음 투입하면 매번 똑같은 작업을 하죠?

pc반입 부터 프로젝트 세팅까지...

거의 공통적으로 매번 반복하는 작업입니다.

물론 pc반입, 파일반입, 방화벽, 권한요청 같은것들은 어쩔 수 없다 치더라도, 
프론트앤드 개발을 위한 기본 골격작업인, 폴더구조나 공통함수, 유틸, api통신모듈, 상태관리, 라우터 등 
공통으로 사용하는 것들은 맨날 반복해서 처음부터 세팅할 필요가 있을까? 라는 생각을 했습니다.



## 4. 하지만 중요한 골뼈대 작업.

그렇다고 해서 이 프론트앤드 기본 뼈대 작업을 대충할 수는 없습니다.
기본 뼈대가 탄탄하게 구성 되어 있어야, 그 위에서 화면구현, 비즈니스 로직 작업을 하고, 나중에는 유지보수에도 문제없을 것입니다.




## 5. 그래서 원하는건

처음부터 공통적인 요소가 모두 구현되어 있고, 검증된 폴더구조로 만들어져있으면 좋겠다. 라고 생각했습니다.



## 6. 이렇게 기반코드가 미리 다 갖춰져 있다면

프로젝트 초반 기반 세팅하고 공통작업하는 시간을 줄여서,
이렇게 Scaffold 기반코드 세팅만으로, 
실제 프론트앤드 개발할 시간을 충분히 벌 수 있을것입니다. 



## 7. 그리고 실제 프론트앤드 개발 시에도

화면개발, 비즈니스로직 외에 
잡다한 여러가지 작업을 Scaffold가 어느정도 제공함으로써 
조금이라도 더 화면 개발에 집중할 수 있게 도울 수 있을 것입니다.



## 8. 그래서 이 react-app-scaffold 를 만들었습니다.


## 9. 이 react-app-scaffold는

react,
typescript,
vite
기반으로 만들었구요,

다음과 같은 구성으로 되어있습니다.

각종 유틸리티를 제공합니다.

실제 사용해보고, 해당 코드를 바로 확인합니다.
그리고 좀 더 디테일한 사용 가이드를 가이드 사이트 형태로 제공합니다.
폴더구조 또한,
많은 개발자들이 병렬로 개발하더라도, 최대한 충돌없이 개발 가능한 폴더구조로 구성되어 있습니다.



## 10. 먼저 UI 컴포넌트는
'@axiom/components/ui' 에서 모든 컴포넌트를 import해서 사용합니다.
만약 필요한 UI 컴포넌트가 더 생긴다면 이곳에 계속 추가할 예정입니다.


## 11. 다음은 유틸리티 함수를 제공합니다.
scaffold에서 제공하는 유틸리티 함수는 import도 필요없이
한줄 코드로 쉽게 사용할 수 있게 제공합니다.

만약에 금융권에서 아주 많이 사용하는 금액표기 콤마 찍기가 필요하다면,
이와같이 유틸리티 comma함수로 쉽게 사용할 수 있습니다.


## 12. 다음으로 커스텀 훅 인데요.

일반적으로 화면개발할 때 가장 많이 사용하는 로직이 아마 서버 api 호출해서, response 데이터를 사용하는 것일 겁니다.
그런데 그게 단순히 api호출에서 끝나는게 아니고,
api호출하고,
로딩처리하고,
실제 가져온 데이터를 상태관리하고,
거기에 또 에러처리까지 해야합니다.

이렇게 여러가지 작업들을 일일이 구현하는것은 오래걸리고 힘든 작업이 될 수 있습니다.

그래서 좀 더 그것들을 쉽게 처리할 수 있게 "useApi 훅"을 제공합니다.
이 useApi 훅은
로딩처리,
에러처리,
상태관리
를 쉽게 처리할 수 있게 합니다.

이와같이 로직 처리할 때 좀 더 편리함을 제공하는 커스텀 훅을 scaffold에서 제공할 것입니다.
