목표
통일된 사용구조 · 개발자 편의 · 쉬운 사용법
DDD 도메인 기반 폴더구조
src/
├─ core/        # 공통 코어 (공유)
├─ domains/     # 업무 도메인 ◄ 분리 단위
│  ├─ 업무1/
│  ├─ 업무2/     # components·pages·router·types
│  └─ …          # 도메인별 독립
├─ publishing/  # 퍼블리셔 작업
└─ shared/      # 전역 공유

업무별로 개발 영역이 격리되어, 여러 도메인을 동시에 작업해도 코드 충돌을 최소화합니다.

01

UI 컴포넌트

02

각종 유틸리티

전역 $util · $router · $ui 와 커스텀 훅으로 반복 로직을 바로 호출.

03

인터랙티브 데모

컴포넌트를 직접 조작해보며 동작과 옵션을 확인 — 학습 곡선을 낮춥니다.
